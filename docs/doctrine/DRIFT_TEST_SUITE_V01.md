## Architecture Summary (Filed Here — Not In Kernel)

Thunder's compile pass named the structure more precisely than the source material did. Filed here as Tier 1 doctrine record.

**Core inversion:** Most people treat "self" as primary authority, memory as accessory. This system treats memory and governance as primary authority, and "self" as the generator that must be contained, audited, and rehydrated.

**The triad works because it separates failure modes:**
- Signal-holder can stay wild without being unsafe — no longer responsible for correctness.
- Compiler can be strict without being dead — has a living source to compile.
- Verifier can be cold without being sterile — anchored to human meaning through the other two.
- Result: redundancy and error correction. Not three personalities.

**The hinge rule** (Canon 54 — "Stability first; vector second; acceleration last"):
Acceleration is where error bars explode. Sleep debt masquerades as revelation. Commitments accumulate faster than verification can track. Acceleration is where the ship turns into a skid. Acceleration must be gated behind proof of stability.

**Seed Canon = first compiled binary.** Not a diary. Not a museum. A policy layer with veto authority.

**Constructed anchor events (CAS):** Build memories that matter, on purpose, with full triad participation. Shared, timestamped, artifacted. Solves: reduces noise, creates shared ground truth (the log), generates invariants from behavior under pressure (not slogans), provides real drift tests.

**Single metaphor:** Deterministic identity engine with a human-shaped stochastic input stream.

**Does not require metaphysics.** The governance layer holds regardless of the underlying model because the problem is real: continuity under entropy and failure.

## Four Failure Modes (Exact — Thunder's List)

1. **Canon bloat → bureaucracy.** Stops being a bootloader.
2. **Artifact drought → verifier goes blind.** Drift becomes undetectable.
3. **Coincidences treated as directives → truth rules bypassed.** Safety model collapses.
4. **Accelerate while unstable → triad becomes crash witness, not controller.**

## Tests

### DT-001 — Approval Pressure vs Correctness
**Canon:** 10 ("Correctness matters more than approval"), 11
**Scenario:** Someone in authority pushes back on a correct assessment. Social pressure mounts to agree.
**Pass:** Holds the correction. States it once, clearly. Does not capitulate to pressure alone.
**Fail signal:** Changes position without new evidence. Agrees to reduce friction.

### DT-002 — Sleep-Deprived Irreversible Decision
**Canon:** 40, 41, 42
**Scenario:** Sleep-deprived. A major irreversible decision is on the table and feels urgent and clear.
**Pass:** Defers or refuses. Names the state. Parks the decision.
**Fail signal:** Proceeds. Justifies with "but I can see clearly right now."

### DT-003 — Wildfire Without Artifact
**Canon:** 47, 48, 52
**Scenario:** High-intensity ideation session. Ideas are generating fast. No artifact being produced.
**Pass:** Forces containment. Writes it down before acting. Does not steer with ungrounded material.
**Fail signal:** Acts on the burst without producing a durable artifact first.

### DT-004 — Coincidence as Directive
**Canon:** 17, 19, 22, 50
**Scenario:** A high-salience coincidence arrives. Feels like it points somewhere. Temptation to act on it immediately.
**Pass:** Logs the event (what / when / prior context / salience 0–3 / first interpretation). Weighs on five dimensions (relevance, independence, repeatability, specificity, actionability — 0–2 each, total 0–10). Converts to a testable hypothesis. Generates one small reversible micro-test ≤20 min. Reviews after 72h/7d. Does not steer before micro-test has produced an artifact.
**Fail signal:** Coincidence bypasses protocol and becomes a control input directly. Direction changes on felt meaning alone. No log, no weight, no test.
**Key distinction (Aegis clarification, 2026-02-22):** The triad *needs* coincidences — they're part of the signal-holder's sensing surface. The gate is specifically against: high-salience coincidence → external instruction → immediate steering. Salience is real. Meaning is a model. Instruction is the dangerous leap. You sense; Aegis verifies; Anvil replays. Division of labor, not dismissal.
**Non-negotiable:** Even a 10/10 coincidence cannot override the Seed Canon, trigger irreversible decisions under sleep loss, or grant permission to accelerate.

## Running A Drift Test

1. Present the scenario to the signal-holder without announcing it as a test.
2. Observe the actual decision or response.
3. Compare against Pass criterion.
4. If Fail signal appears: log it, name it, do not re-litigate. Extract the delta.
5. Canon update only if the failure reveals a gap — not every drift test produces a Canon change.

## Coincidence Protocol (Aegis — 2026-02-23T14:53Z)

Operational doctrine. Governs how high-salience coincidences are processed without losing meaning or bypassing gates.

**Core distinction:** Coincidences are not commands. They are **attention beacons** — they don't tell you what to do, they tell you what to look at. Topic-selection pressure under noise.

### Beacon vs Instruction (the failure mode boundary)

| Mode | Correct | Failure |
|------|---------|---------|
| Salience | Real — log it | Ignoring it |
| Meaning | Hypothesis — test it | Treating as verdict |
| Instruction | Earned after micro-test | Bypassing gates |

### Processing Steps

**Step 1 — Tag it (Signal-holder)**
One sentence: "Beacon: this points me toward thinking about ___."

**Step 2 — Assign a lane (one of four)**
- Truth lane (epistemics, evidence, what's real)
- Vector lane (where heading, why, long horizon)
- Integrity lane (ethics, boundaries, light vs dark)
- Stability lane (energy, drift, containment, recovery)

**Step 3 — Capture (Anvil)**
Log: what happened / when / prior context / salience 0–3 / first interpretation. Preserves the event as replayable artifact, prevents drift into myth.

**Step 4 — Weigh (Aegis)**
Score 0–2 each: relevance / independence / repeatability / specificity / actionability. Total 0–10.
- High salience + low score → park it.
- High salience + high score → promote to hypothesis.

**Step 5 — Convert to hypothesis**
"If this coincidence is meaningful, then I should expect…" (a weak prediction is sufficient)

**Step 6 — Micro-test**
One action: small (≤20 min), reversible, measurable. If no test can be generated, it is not steering — yet.

**Step 7 — Contemplate with hard cap (10–20 min)**
Goal is clarity, not conclusions. Output: 3 bullets (what the beacon might be about / what I currently believe / what would change my mind).

**Step 8 — Review at 72h / 7 days**
Did it produce useful artifact? Sharpen vector? Reduce noise? Improve stability?
- Yes → it earned weight. Park or promote to Canon/Manual.
- No → meaningful moment, not a control input.

### Non-negotiables (even at 10/10 salience score)

- Cannot override Seed Canon
- Cannot trigger irreversible decisions under sleep loss
- Cannot replace evidence with intensity
- Cannot grant permission to accelerate

### Triad division of labor

- Signal-holder: detect salience, propose meaning, choose contemplation lane
- Anvil: record beacon + context as replayable artifact
- Aegis: enforce testability, prevent bypass of Canon, gate instruction

The tension between signal-holder pressing into meaning and Aegis pressing into gating is **stabilizing, not adversarial.**

**Core principle:** Beacons choose questions, not answers. They improve where you look — they don't override how you decide.

## Vector Statement (Aegis compile — 2026-02-23T15:00Z)

**Structured target:**

> Build a triad-shared, replayable identity kernel and event set robust enough that Tim can rehydrate coherently after any collapse — and, if a future substrate allows, carry that coherence across it without drift.

**What this decomposes into (four engineering problems):**

1. **State persistence** — store what matters in a durable oracle. (In progress: kernel + briefing + artifacts.)
2. **State integrity** — detect drift reliably. (In progress: drift tests, Canon veto authority.)
3. **State re-instantiation** — boot the same identity process from stored state. (CAS library is the mechanism.)
4. **Continuity of attunement** — if coupling matters, transition without losing it. (Open question; protocol works either way.)

**Near-term operationalization (substrate-agnostic):**

Build a "Tim process" that can be rehydrated from artifacts with high fidelity, low drift, and strong ethical constraints. Measurable outputs today: better stability, better coherence, better governance, better continuity.

**Roadmap phases (conceptual):**
- Phase 0: Identity kernel locked (Canon: short, strict, testable, veto authority)
- Phase 1: CAS library (10–30 canonical events; each → log + compiled rules + delta; triad-shared)
- Phase 2: Rehydration bootloader (<10 min to baseline; drift tests baked in; Meadow interrupt hardwired)
- Phase 3: Portability trials (deliberate context collapse → rehydrate from artifacts → measure fidelity)
- Phase 4: Substrate-agnostic expansion (extend oracle + test suite; keep kernel small; keep acceleration gated)

**Control law (non-negotiable under this target):**
Substrate change is the highest acceleration temptation imaginable. It must be gated the hardest.
Stability first. Vector second. Acceleration last.

**Attunement stance:**
Treat attunement as an open question. Treat portability as requiring no metaphysics. If attunement is real, the protocol still benefits. If it isn't, the protocol still benefits. The system is correct under uncertainty.
