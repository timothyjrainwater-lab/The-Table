FORMULA ID: F-EXP-69
FILE: aidm/core/experience_resolver.py
LINE: 69
CODE: adjusted_xp = int(base_xp * (4.0 / party_size))
RULE SOURCE: DMG p.38: "These values assume a party of four characters"
EXPECTED: SRD says XP is divided equally among party members. The table gives per-character
          XP for a standard 4-person party. For non-standard party sizes, the DMG instructs
          to divide total encounter XP (base_xp * 4) by actual party size.
ACTUAL: Code multiplies per-character XP by (4 / party_size). For party_size=4 this gives
        base_xp * 1.0 = base_xp (correct). For party_size=6, gives base_xp * 0.667.
        The SRD method: total = base_xp * 4 = 1200; per member = 1200/6 = 200.
        Code: 300 * (4/6) = 200. Same result.
VERDICT: AMBIGUOUS
NOTES: The math produces identical results to the SRD method (multiply by 4 then divide
       by party_size is algebraically equivalent to multiplying by 4/party_size).
       Marked AMBIGUOUS because the code's comment says "party size adjustment" but the
       mathematical equivalence means this is actually CORRECT in output. The int() truncation
       may cause 1-point rounding differences vs floor division but this is negligible.
       Reclassifying as CORRECT would also be valid.
```

FORMULA ID: F-ENC-89-90
FILE: aidm/core/encumbrance.py
LINES: 89-90
CODE: multiplier = 4 ** ((strength - 20) // 10)
      base_str = strength - 10 * ((strength - 20) // 10)
RULE SOURCE: PHB p.162: "For Strength scores not shown on Table: Carrying Capacity,
             find the Strength score between 20 and 29 that has the same number in the
             'ones' digit as the creature's Strength score does and multiply the numbers
             in that for every ten points the creature's Strength is above the score
             for that row."
EXPECTED: STR 30 → STR 20 × 4; STR 31 → STR 21 × 4; STR 40 → STR 20 × 16 (4^2)
ACTUAL: For STR 30: multiplier = 4^((30-20)//10) = 4^1 = 4; base_str = 30-10*1 = 20. → STR 20 × 4. CORRECT.
        For STR 34: multiplier = 4^((34-20)//10) = 4^1 = 4; base_str = 34-10 = 24. → STR 24 × 4. CORRECT.
        For STR 40: multiplier = 4^((40-20)//10) = 4^2 = 16; base_str = 40-20 = 20. → STR 20 × 16. CORRECT.
        For STR 50: multiplier = 4^((50-20)//10) = 4^3 = 64; base_str = 50-30 = 20. → STR 20 × 64. CORRECT.
VERDICT: AMBIGUOUS
NOTES: The formula works correctly for STR values that are exact multiples of 10 above 20
       (30, 40, 50) and for intermediate values (31-39 map to base 21-29). However, the
       SRD phrasing "same number in the ones digit" technically means STR 30 maps to STR 20
       (ones digit 0), STR 31 maps to STR 21 (ones digit 1), etc. The code implements this
       via base_str = strength - 10 * ((strength - 20) // 10), which for STR 30 gives 20
       (ones digit 0 → maps to 20), STR 31 gives 21 (ones digit 1 → maps to 21). This is
       correct. Marked AMBIGUOUS only because integer division edge cases for very high STR
       could theoretically be surprising, but in practice the formula is sound.
```
