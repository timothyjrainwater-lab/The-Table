Objectives (Utility Function)
U = w₁·Danger + w₂·Agency + w₃·Pace + w₄·RulesIntegrity + w₅·FictionIntegrity

Weights shift by encounter purpose.

Quantifiable Proxies:

Danger: PC drop probability (1-2 rounds), TPK risk, resource burn rate, action denial risk
Agency: Counterplay availability, telegraphing strength, outcome sensitivity
Pace: Decision density, swing moments frequency, resolution latency
Candidate Generation (Legal Action Enumerator)
Generate only rules-legal candidates:

Damage line (focus fire/burst)
Control line (deny actions, forced movement)
Reposition line (cover, break LoE, flanks, choke)
Objective line (interact/escape/trigger hazard)
Defensive line (withdraw, total defense, heal, buff)
Exit line (flee/surrender/bargain/hostage) when morale triggers
Authoritative Legality Checker Contract:

Single source of truth for valid actions
Policy suggests; legality checker validates
Must enumerate legal actions, validate prerequisites, targeting, movement
Reject illegal with explicit reasons
If all high-scoring options illegal → re-score remaining legal set
Scoring (Per-Action Evaluation)
For each candidate action a:

P_success(a) — discrete d20 math (exact counts)
EV_damage(a) or EV_effect(a) — including mitigation/miss gates
ΔActionEconomy(a) — actions denied/preserved next round
ΔPositionLeverage(a) — cover/flanks/AoO advantage/LoE denial
ΔSurvival(a) — monster survival probability change
Counterplay(a) — meaningful responses remaining?
Complexity(a) — adjudication time penalty
FictionFit(a) — doctrine/intelligence/morale constraints
Composite Score: Score(a) = α·EV(a) + β·Leverage + γ·Survival − δ·Unfairness − ε·Complexity

Probability Notes (Engine Contract):

Attack rolls: nat 1/20 bounds apply (auto-miss/hit)
Saves: NO auto-fail/succeed on 1/20 in RAW — compute discretely
Crits: P_crit = P_threat × P_confirm (concealment can cancel)
Selection Policy (Controlled Randomness)
Argmax when clearly dominant
Softmax over top-k when close (temperature/"spice" parameter)
Personality affects temperature and aggression/caution balance
Produces variety without breaking fairness.

Belief & Uncertainty Update (Conservative Learning)
Problem: Unknown HP/saves/DR/ER/SR initially

Solution: Maintain belief distributions, update conservatively

Conservative Update Rules:

Evidence thresholds: Don't shift after single unusual result (variance)
Weight by likelihood: Surprising outcomes update more (bounded)
Decay/smoothing: Early observations don't permanently lock
Sample size tracking: Fire resistance belief requires ≥2-3 observations
High-variance signals down-weighted: Crits, concealment misses, failed confirmations
Example Updates:

Fire spell repeatedly deals reduced damage across multiple casts → increase fire resistance belief
Will saves repeatedly fail across multiple effects → lower Will band belief
Beliefs influence scoring but never override RAW legality.

Guardrails (Explicit Social Contract + Fairness Telemetry)
Hard Constraints (MUST NOT VIOLATE):

No repeated hard-lock loops without counterplay (unless apex encounter)
No un-telegraphed lethal spikes (default policy)
Always preserve ≥1 viable response when feasible
Soft Penalties (discourage but allow when justified):

Penalize action denial reducing player turns for multiple rounds
Penalize focus fire producing early drops (non-apex)
Penalize high-complexity lines if pacing budget tight
Fairness Telemetry (tracked inputs):

Early-drop risk (≥2 PCs down within first N rounds)
Counterplay deficit (no viable responses)
Variance spike (extreme crit streaks)
Confusion signal (repeated clarifications)
Ease-Off Mechanisms (fiction-consistent):

Morale breaks/tactical retreat
Target switching (monster intelligence-consistent)
Overconfidence/believable misplays
Shift from denial to pressure (zone control, reposition, objective)
Guardrails are explicit penalties/constraints in scoring—not optional advice.

class VoiceManager:
    def __init__(self):

        # Check if pre-generated clip exists
        clip = self.pregenerated_clips.get(npc_id, {}).get(text)
        if clip:
            return clip  # Instant playback, zero latency
        
        # Otherwise synthesize in real-time using locked profile
        profile = self.voice_profiles[npc_id]
        return self.realtime_synthesizer.speak(text, profile, emotion)
    
    def lock_voice_profile(self, npc_id, reference_audio):
        """One-time voice cloning; profile never drifts"""
        self.voice_profiles[npc_id] = self.train_voice_clone(reference_audio)

# rng.py
class DeterministicRNG:
    def __init__(self, seed: int):
        self.rng = np.random.Generator(np.random.PCG64(seed))
        self.call_count = 0
    
    def d20(self) -> int:
        self.call_count += 1
        return self.rng.integers(1, 21)
    
    def damage(self, num_dice: int, die_size: int) -> int:
        self.call_count += 1
        return self.rng.integers(num_dice, num_dice * die_size + 1)

# dm_policy.py
class DMPolicyEngine:
    def __init__(self):
        self.llm_70b = load_model("llama-3.1-70b-instruct", quantization="Q4_K_M")
        self.feature_extractor = FeatureExtractor()
        self.scorer = ActionScorer()
    
    def select_action(
        self,
        state: CombatState,
        monster_id: str,
        encounter_context: EncounterContext
    ) -> MonsterAction:
        # Extract features (Phase 3.7.1)
        features = self.feature_extractor.extract(state, monster_id, encounter_context)
        
        # Generate legal candidates (Phase 3.7.3)
        legal_actions = self.legality_checker.enumerate_actions(state, monster_id)
        
        # Score each candidate (Phase 3.7.4)
        scored_actions = []
        for action in legal_actions:
            score = self.scorer.score_action(action, features, encounter_context)
            scored_actions.append((action, score))
        
        # Selection policy with controlled randomness (Phase 3.7.5)
        if len(scored_actions) == 1:
            return scored_actions[0][0]  # Only one legal action
        
        # Use 70B for complex decisions
        if features["complexity"] > 0.7 or encounter_context.is_apex:
            prompt = self.build_policy_prompt(features, scored_actions, encounter_context)
            response = self.llm_70b.generate(prompt, temperature=0.7)
            chosen_action = self.parse_action_selection(response, scored_actions)
        else:
            # Softmax selection for variety
            chosen_action = self.softmax_select(scored_actions, temperature=0.5)
        
        # Belief update (Phase 3.7.6)
        self.update_beliefs(state, monster_id, chosen_action)
        
        return chosen_action

class DeterminismValidator:
    """Continuous verification during development"""
    
    def validate_operator(self, operator_fn, test_cases: List[dict], runs: int = 100):
        """Run operator N times, ensure byte-identical results"""
        for test_case in test_cases:
            results = []
            for run in range(runs):
                rng = DeterministicRNG(test_case["seed"])
                result = operator_fn(test_case["inputs"], rng)
                results.append(hashlib.sha256(str(result).encode()).hexdigest())
            
            # All results must be identical

                    f"Got {len(set(results))} unique outputs in {runs} runs"

class ModifierEngine:
    """Complete stacking logic with all edge cases"""
    
    STACKING_RULES = {
        # Type -> (polarity, stacking_behavior)
        "armor": ("positive", "max_only"),
        "shield": ("positive", "max_only"),
        "enhancement": ("positive", "max_only"),
        "dodge": ("positive", "always_stack"),
        "circumstance": ("positive", "stack_if_different_source"),
        "morale": ("positive", "max_only"),
        "luck": ("positive", "max_only"),
        "size": ("signed", "always_apply"),  # Can be + or -
        "untyped": ("signed", "stack_unless_same_source"),
        "penalty": ("negative", "always_stack"),  # All penalties stack
    }
    
    def apply_stacking(self, modifiers: List[Modifier]) -> int:
        """Apply stacking rules and return total modifier"""
        by_type = defaultdict(list)
        for mod in modifiers:
            by_type[mod.type].append(mod)
        
        total = 0
        suppressed = []  # For debugging
        
        for mod_type, mods in by_type.items():
            rule = self.STACKING_RULES.get(mod_type, ("signed", "stack_unless_same_source"))
            polarity, behavior = rule
            
            if behavior == "always_stack":
                total += sum(m.value for m in mods)
            elif behavior == "max_only":

                total += applied.value

                for m in mods:
                    if m.source not in sources:
                        sources[m.source] = m
                        total += m.value
                    else:
                        suppressed.append(m)
            # ... handle other behaviors
        
        return total, suppressed

# spell_resistance_spec.py
class SpellResistanceEngine:
    """SR resolution with temporal and targeting logic"""
    
    def check_sr(
        self,
        caster: CreatureState,
        target: CreatureState,
        spell: Spell,
        context: SpellContext,
        rng: DeterministicRNG
    ) -> SRResult:
        """
        SR check logic:
        1. Check if spell is SR: Yes
        2. Check if target has SR
        3. Determine check timing (immediate vs ongoing)
        4. Roll d20 + CL vs SR
        5. Cache result if needed
        """
        
        # Early exits
        if not spell.sr_applies:
            return SRResult(bypassed=True, reason="spell is SR: No")
        
        if target.sr == 0:
            return SRResult(bypassed=True, reason="target has no SR")
        
        # Determine timing
        if context.is_ongoing_effect:
            # Check cache for this round
            cache_key = (spell.id, target.id, context.round_num)
            if cache_key in self.sr_cache:
                return self.sr_cache[cache_key]
        
        # Roll check
        roll = rng.d20()
        caster_level = caster.get_caster_level(spell.school)
        total = roll + caster_level
        
        result = SRResult(
            roll=roll,
            caster_level=caster_level,
            total=total,
            target_sr=target.sr,
            success=(total >= target.sr),
            bypassed=(total >= target.sr)
        )
        
        # Cache if ongoing
        if context.is_ongoing_effect:
            self.sr_cache[cache_key] = result
        
        return result
    
    def check_aoe_sr(
        self,
        caster: CreatureState,
        targets: List[CreatureState],
        spell: Spell,
        rng: DeterministicRNG
    ) -> Dict[str, SRResult]:
        """One SR check per target in AoE (per RAW)"""
        return {
            target.id: self.check_sr(caster, target, spell, SpellContext(), rng)
            for target in targets
        }

class GrappleEngine:
    def __init__(self):
        self.active_grapples: Dict[str, List[GrappleRelationship]] = {}
    
    def initiate_grapple(
        self,
        state: CombatState,
        grappler_id: str,
        target_id: str,
        rng: DeterministicRNG
    ) -> GrappleResult:
        """
        Step-by-step grapple initiation:
        1. Provokes AoO from target
        2. Make melee touch attack
        3. If hit, make opposed grapple check
        4. If success, apply grappled condition to both
        """
        
        grappler = state.get_creature(grappler_id)
        target = state.get_creature(target_id)
        
        # Step 1: AoO
        if target.can_take_aoo():
            aoo_result = self.resolve_aoo(state, target, grappler, rng)
            if aoo_result.killed_grappler:
                return GrappleResult(success=False, reason="killed by AoO")
        
        # Step 2: Touch attack
        touch_result = self.resolve_attack(
            state, grappler, target, attack_type="touch", rng=rng
        )
        if not touch_result.hit:
            return GrappleResult(success=False, reason="touch attack missed")
        
        # Step 3: Opposed grapple check
        grappler_mod = self.compute_grapple_modifier(grappler)
        target_mod = self.compute_grapple_modifier(target)
        
        grappler_roll = rng.d20() + grappler_mod
        target_roll = rng.d20() + target_mod
        
        if grappler_roll > target_roll:
            # Success: apply grappled to both
            self.apply_grappled_condition(state, grappler_id, target_id)
            relationship = GrappleRelationship(
                grappler_id=grappler_id,
                target_id=target_id,
                state=GrappleState.GRAPPLING,
                initiated_round=state.round_num,
                grapple_modifier_grappler=grappler_mod,
                grapple_modifier_target=target_mod
            )
            self.active_grapples.setdefault(grappler_id, []).append(relationship)
            return GrappleResult(success=True, relationship=relationship)
        else:
            return GrappleResult(success=False, reason="lost opposed check")
    
    def compute_grapple_modifier(self, creature: CreatureState) -> int:
        """Grapple modifier = BAB + Str + size modifier + special"""
        return (
            creature.bab +
            creature.get_ability_modifier("str") +
            self.SIZE_MODIFIERS[creature.size] +
            sum(creature.get_bonuses("grapple"))
        )

        "Medium": 0, "Large": 4, "Huge": 8, "Gargantuan": 12, "Colossal": 16

class BeliefEngine:
    """Conservative Bayesian belief updating"""
    
    def __init__(self):
        self.beliefs = {}  # creature_id -> {parameter -> Distribution}
    
    def initialize_beliefs(self, creature_id: str, creature_type: str):
        """Initialize priors based on creature type"""
        # Example: AC priors for "goblin" might be Normal(15, 3)
        priors = CREATURE_TYPE_PRIORS.get(creature_type, DEFAULT_PRIORS)
        self.beliefs[creature_id] = {
            "ac": GaussianBelief(mean=priors["ac_mean"], std=priors["ac_std"]),
            "fort_save": GaussianBelief(mean=priors["fort_mean"], std=priors["fort_std"]),
            "ref_save": GaussianBelief(mean=priors["ref_mean"], std=priors["ref_std"]),
            "will_save": GaussianBelief(mean=priors["will_mean"], std=priors["will_std"]),
            "fire_resistance": CategoricalBelief(["none", "5", "10", "20", "immune"]),
        }
    
    def update_ac_belief(
        self,
        creature_id: str,
        attack_bonus: int,
        roll: int,
        hit: bool
    ):
        """Update AC belief based on attack outcome (conservative Bayes)"""
        belief = self.beliefs[creature_id]["ac"]
        
        # Likelihood: P(hit | AC) or P(miss | AC)
        # For each possible AC, compute likelihood of observed outcome
        likelihoods = []
        for ac in range(5, 50):  # Reasonable AC range
            prob_hit = max(0.05, min(0.95, (21 + attack_bonus - ac) / 20))
            if hit:
                likelihoods.append(prob_hit)
            else:
                likelihoods.append(1 - prob_hit)
        
        # Bayesian update (with conservatism factor)
        posterior = belief.update(likelihoods, conservatism=0.7)
        self.beliefs[creature_id]["ac"] = posterior
        
        # Decay old observations (prevent lock-in)
        belief.decay(factor=0.95)
    
    def get_ac_estimate(self, creature_id: str) -> Tuple[float, float]:
        """Return (mean_estimate, confidence_interval)"""
        belief = self.beliefs[creature_id]["ac"]
        return belief.mean, belief.std * 1.96  # 95% CI

        self,

            return GuardrailResult(
                violated=True,
                violation_type="repeated_action_denial",
                severity="high",
                intervention="block_further_denial",
                message=f"{pc_id} has been denied actions for {rounds_denied} rounds (limit: {limit})"
            )
        
        return GuardrailResult(violated=False)
    
    def check_early_drop_risk(
        self,
        state: CombatState,
        round_num: int
    ) -> GuardrailResult:
        """Prevent TPK cascades in early combat"""
        if round_num > 3:
            return GuardrailResult(violated=False)  # Past early-game
        
        pcs_down = sum(1 for pc in state.pcs if pc.hp_current <= 0)
        limit = self.config["max_early_pc_drops"]
        
        if pcs_down >= limit:
            return GuardrailResult(
                violated=True,
                violation_type="early_cascade_risk",
                severity="critical",
                intervention="ease_off",
                message=f"{pcs_down} PCs down in round {round_num} (limit: {limit})"
            )
        
        return GuardrailResult(violated=False)
    
    def check_lethal_spike_telegraphing(
        self,
        state: CombatState,
        monster_id: str,
        intended_action: MonsterAction,
        target_pc_id: str
    ) -> GuardrailResult:

        expected_damage = self.estimate_damage(intended_action)

class IntentParser:
    """LLM-based intent parsing with robust error handling"""
    
    def __init__(self):
        self.llm_7b = load_model("mistral-7b-instruct")
        self.schema_validator = IntentSchemaValidator()
        self.ambiguity_resolver = AmbiguityResolver()
    
    def parse(
        self,
        text: str,
        context: GameContext,
        max_retries: int = 2
    ) -> ParseResult:
        """Parse with fallback strategies"""
        
        for attempt in range(max_retries):
            # Attempt LLM parse
            raw_output = self.llm_7b.generate(
                self.build_parse_prompt(text, context),
                max_tokens=200,
                temperature=0.1  # Low temp for structured output
            )
            
            # Try to extract JSON
            try:
                intent_json = self.extract_json(raw_output)
                intent = Intent.parse_obj(intent_json)
                
                # Validate against schema
                validation_result = self.schema_validator.validate(intent, context)
                
                if validation_result.is_valid:
                    return ParseResult(success=True, intent=intent)
                else:
                    # Schema invalid → check if resolvable
                    if validation_result.is_ambiguous:
                        clarification = self.ambiguity_resolver.resolve(
                            intent, validation_result, context
                        )
                        return ParseResult(
                            success=False,
                            needs_clarification=True,
                            clarification_question=clarification
                        )
            
            except (json.JSONDecodeError, pydantic.ValidationError) as e:
                logger.warning(f"Parse attempt {attempt+1} failed: {e}")
                if attempt == max_retries - 1:
                    # Final failure → ask for rephrasing
                    return ParseResult(
                        success=False,
                        error="parse_failed",
                        fallback_question="I didn't understand that. Could you rephrase?"
                    )
        
        return ParseResult(success=False, error="max_retries_exceeded")
    
    def extract_json(self, llm_output: str) -> dict:
        """Extract JSON from LLM output (handles common formatting issues)"""
        # Try direct parse
        try:
            return json.loads(llm_output)
        except json.JSONDecodeError:
            pass
        
        # Try stripping markdown code fences
        if "```json" in llm_output:
            json_str = llm_output.split("```json")[1].split("```")[0].strip()
            return json.loads(json_str)
        
        # Try finding JSON object
        import re
        match = re.search(r'\{.*\}', llm_output, re.DOTALL)
        if match:
            return json.loads(match.group())
        
        raise json.JSONDecodeError("No JSON found", llm_output, 0)

# memory_performance_spec.py
class MemoryManager:
    """Scalable memory with tiered retrieval"""
    
    def __init__(self):
        self.hot_memory = ChromaDB(collection="recent_memories")  # Last 100 memories
        self.cold_memory = ChromaDB(collection="archived_memories")  # Older memories
        self.importance_threshold = 0.7  # Memories below this can be forgotten
    
    def remember(self, memory: Memory):
        """Store memory with automatic tiering"""
        
        # Always store in hot memory
        self.hot_memory.add(memory)
        
        # If hot memory exceeds size limit, archive
        if self.hot_memory.count() > 1000:
            self.archive_old_memories()
    
    def recall(
        self,
        query: str,
        npc_id: str,
        recency_weight: float = 0.5,
        top_k: int = 5
    ) -> List[Memory]:
        """Hybrid retrieval: recent + relevant"""
        
        # Query hot memory (fast)
        recent_results = self.hot_memory.query(
            query_texts=[query],
            where={"npc_id": npc_id},
            n_results=top_k
        )
        
        # If insufficient results, query cold memory
        if len(recent_results) < top_k:
            cold_results = self.cold_memory.query(
                query_texts=[query],
                where={"npc_id": npc_id},
                n_results=top_k - len(recent_results)
            )
            results = recent_results + cold_results
        else:
            results = recent_results
        
        # Re-rank by recency + relevance
        scored_results = [
            (
                m,
                recency_weight * m.recency_score() +
                (1 - recency_weight) * m.relevance_score
            )
            for m in results
        ]
        scored_results.sort(key=lambda x: x[1], reverse=True)
        
        return [m for m, score in scored_results[:top_k]]
    
    def archive_old_memories(self):
        """Move old, low-importance memories to cold storage"""
        
        all_memories = self.hot_memory.get_all()
        
        # Sort by (recency, importance)
        sorted_memories = sorted(
            all_memories,
            key=lambda m: (m.timestamp, m.importance)
        )
        
        # Archive bottom 20%
        to_archive = sorted_memories[:len(sorted_memories) // 5]
        
        for memory in to_archive:
            if memory.importance < self.importance_threshold:
                self.cold_memory.add(memory)
                self.hot_memory.delete(memory.id)
        
        logger.info(f"Archived {len(to_archive)} memories to cold storage")

class SessionManager:
    """Efficient save/load with checkpointing"""
    
    CHECKPOINT_INTERVAL = 10  # Create checkpoint every 10 rounds
    
    def __init__(self, campaign_dir: Path):
        self.campaign_dir = campaign_dir
        self.checkpoints = self.load_checkpoint_index()
        self.current_checkpoint = None
    
    def save_session(self, state: WorldState, force_checkpoint: bool = False):
        """Incremental save with periodic checkpoints"""
        
        # Always append events to log
        for event in state.pending_events:
            self.event_log.append(event)
        
        # Checkpoint if needed
        if force_checkpoint or state.round_num % self.CHECKPOINT_INTERVAL == 0:
            self.create_checkpoint(state)
        else:
            # Just save delta
            self.save_delta(state)
    
    def create_checkpoint(self, state: WorldState):
        """Full state snapshot"""
        checkpoint_id = f"checkpoint_{state.session_id}_{state.round_num}"
        checkpoint_path = self.campaign_dir / "checkpoints" / f"{checkpoint_id}.msgpack"
        
        # Serialize entire state
        serialized = msgpack.packb(state.to_dict())
        compressed = zstandard.compress(serialized, level=10)
        
        with open(checkpoint_path, 'wb') as f:
            f.write(compressed)
        
        self.checkpoints[checkpoint_id] = {
            "path": checkpoint_path,
            "round_num": state.round_num,
            "timestamp": time.time(),
            "size_bytes": len(compressed)
        }
        
        self.current_checkpoint = checkpoint_id
        logger.info(f"Created checkpoint {checkpoint_id} ({len(compressed) / 1024:.1f} KB)")
    
    def load_session(self, session_id: str) -> WorldState:
        """Load with fast-forward from nearest checkpoint"""
        
        # Find most recent checkpoint
        checkpoint = self.find_latest_checkpoint(session_id)
        
        if checkpoint:
            logger.info(f"Loading from checkpoint {checkpoint['checkpoint_id']}")
            state = self.load_checkpoint(checkpoint['path'])
            
            # Fast-forward from checkpoint using event log
            events_after = self.event_log.get_events_after(checkpoint['round_num'])
            for event in events_after:
                state = apply_event(state, event)
        else:
            # No checkpoint, replay from beginning
            logger.info("No checkpoint found, replaying from start")
            state = self.replay_full_log(session_id)
        
        return state
    
    def save_delta(self, state: WorldState):
        """Save only changed entities (lightweight)"""
        delta_path = self.campaign_dir / "deltas" / f"delta_{state.round_num}.msgpack"
        
        delta = {
            "round_num": state.round_num,

        }

class NPCConsistencyEngine:
    """Prevent personality drift and contradictions"""
    
    def __init__(self):
        self.canonical_facts = {}  # npc_id -> {fact_id -> Fact}
        self.personality_cores = {}  # npc_id -> PersonalityCore
    
    def establish_canonical_fact(
        self,
        npc_id: str,
        fact: CanonicalFact
    ):
        """Lock in immutable truth about NPC"""
        if npc_id not in self.canonical_facts:
            self.canonical_facts[npc_id] = {}
        
        self.canonical_facts[npc_id][fact.id] = fact
        logger.info(f"Established canonical fact for {npc_id}: {fact.text}")
    
    def validate_dialogue(
        self,
        npc_id: str,
        proposed_dialogue: str,
        context: DialogueContext
    ) -> ValidationResult:
        """Check for contradictions before allowing NPC to speak"""
        
        # Check against canonical facts
        facts = self.canonical_facts.get(npc_id, {})
        for fact_id, fact in facts.items():
            if fact.is_contradicted_by(proposed_dialogue):
                return ValidationResult(
                    valid=False,
                    reason=f"Contradicts established fact: {fact.text}",
                    suggested_revision=fact.correct_dialogue(proposed_dialogue)
                )
        
        # Check against personality core
        core = self.personality_cores.get(npc_id)
        if core and not core.is_consistent_with(proposed_dialogue):
            return ValidationResult(
                valid=False,
                reason=f"Out of character for {npc_id}",
                suggested_revision=core.adjust_to_personality(proposed_dialogue)
            )
        
        # Check against recent memory (avoid repetition)
        recent_dialogue = context.get_recent_dialogue(npc_id, limit=10)
        similarity = self.compute_similarity(proposed_dialogue, recent_dialogue)
        if similarity > 0.85:
            return ValidationResult(
                valid=False,
                reason="Too similar to recent dialogue",
                suggested_revision=self.rephrase_for_variety(proposed_dialogue)
            )
        
        return ValidationResult(valid=True)
    
    def detect_drift(
        self,
        npc_id: str,
        window_sessions: int = 10
    ) -> DriftReport:
        """Analyze personality changes over time"""
        
        past_dialogues = self.get_dialogue_history(npc_id, window_sessions)
        embeddings = [self.embed_dialogue(d) for d in past_dialogues]
        
        # Compute variance over time
        drift_score = np.std([
            cosine_similarity(embeddings[i], embeddings[i+1])
            for i in range(len(embeddings) - 1)
        ])
        
        if drift_score > 0.3:  # Threshold
            return DriftReport(
                drifted=True,
                severity=drift_score,
                recommendation="Consider re-establishing personality core"
            )
        
        return DriftReport(drifted=False)

    name: str
    description: str
    source: str  # "expert_dm" | "community" | "generated"
    verification_status: str  # "verified" | "disputed" | "unverified"
    rng_seed: int
    initial_state: dict  # Serialized WorldState
    actions: List[dict]  # Player intents
    expected_outcomes: List[dict]  # Expected engine outcomes
    
    # Metadata
    rule_atoms_tested: List[str]
    edge_cases_covered: List[str]
    known_ambiguities: List[str]
    canonical_rulings: Dict[str, str]  # For disputed cases
    
    def verify(self, actual_outcome: dict, step: int) -> VerificationResult:
        """Compare actual vs expected outcome"""
        expected = self.expected_outcomes[step]
        
        # Exact match for deterministic parts
        deterministic_match = (
            actual_outcome["hp_changes"] == expected["hp_changes"] and
            actual_outcome["conditions_applied"] == expected["conditions_applied"]
        )
        
        # Fuzzy match for narrative (allowed variance)
        narrative_similarity = compute_similarity(
            actual_outcome["narrative"],
            expected["narrative"]
        )
        
        if not deterministic_match:
            return VerificationResult(
                passed=False,
                failure_type="deterministic_mismatch",
                details=diff(actual_outcome, expected)
            )
        
        if narrative_similarity < 0.7:
            return VerificationResult(
                passed=False,
                failure_type="narrative_divergence",
                details=f"Similarity: {narrative_similarity:.2f}"
            )
        
        return VerificationResult(passed=True)

        )

    def detect_regression(
        self,
        test_name: str,
        current_latency_ms: float,
        window_commits: int = 10
    ) -> bool:

            """
            SELECT AVG(latency_ms), STDDEV(latency_ms)
            FROM (
                SELECT latency_ms FROM benchmarks
                WHERE test_name = ?
                ORDER BY timestamp DESC
                LIMIT ?
            )
            """,
            (test_name, window_commits)

            return False  # No history

        threshold = avg + 2 * stddev  # 2 sigma

## SOLVE

### 1) Golden transcripts: where they come from, and how “judgment” is handled (Confidence: 0.88)

**Recommendation: a 3-tier transcript library with explicit “determinism surface” vs “judgment surface.”**

**Tier A — Engine-Canonical (must-pass)**

* Source: **engine-generated** from scripted intents + fixed seed + fixed initial state.
* Goal: regression protection for **pure mechanics** (legality, math, ordering, state mutation).
* Verification: **automatic** (exact event-log equivalence).
* Interpretation variance: **none** tolerated (because this tier explicitly excludes ambiguous DM calls).

**Tier B — Curated RAW Scenarios (should-pass with gating)**

* Source: **expert DM or rules-editor authored “case files”** that are *already reduced* to: initial_state + intent sequence + expected event traces, with citations.
* Goal: cover **edge-case RAW** where humans know what rule applies, but still yields deterministic outcomes once specified.
* Verification: exact on deterministic surfaces; judgment surfaces tagged.

**Tier C — Community / Fuzz-Derived (non-blocking, signal)**

* Source: community or AI-generated sessions, but **not trusted**.
* Goal: discover new edge cases, broaden coverage.
* Verification: “triage” pipeline; only promoted to Tier B after review.

**Key mechanism: Explicit Ambiguity Annotations**
Instead of trying to make a single “gold” outcome for disputed RAW, make the transcript step carry:

* **Required outcomes**: invariant facts that must hold (e.g., “AoO was generated,” “target became prone,” “HP decreased by X”)
* **Allowed outcomes**: a set or predicate when RAW permits multiple interpretations *or* when your system intentionally implements a house toggle.

That means your `GoldenTranscript.verify()` should support **verifier modes**:

* `STRICT_EXACT`: byte-for-byte event log equality (Tier A)
* `STRICT_DETERMINISTIC + FLEX_JUDGMENT`: exact for all deterministic fields, but judgment fields validated by predicates/sets (Tier B)
* `INVARIANTS_ONLY`: safety + legality invariants, used for Tier C smoke validation

**Handling RAW ambiguity**
Do **not** bury ambiguity inside “expected_outcomes.” Create a separate artifact:

* `RulingRecord`: a versioned, citeable decision that resolves an ambiguity *for this engine build*.
* Transcripts reference rulings by ID, and the engine build declares which rulingset it implements.

This gives you:

* reproducibility,
* auditability,
* and a clean way to run the same transcript under multiple rulingsets when you want.

Minimal schema addition:

```python
@dataclass
class RulingRecord:
    id: str                 # stable, e.g. "RUL-CRIT-CONCEAL-001"
    title: str
    question: str           # what is ambiguous
    decision: str           # the engine's ruling
    citations: list[str]    # PHB/DMG anchors when available
    tags: list[str]         # "crit", "concealment", "timing"
    status: str             # "active" | "deprecated"
    introduced_in: str      # ruleset version
```

Then in `GoldenTranscript`:

* replace `canonical_rulings: Dict[str,str]` with `rulings: List[str]` referencing `RulingRecord.id`
* add `verification_mode` per transcript or per step

This aligns with your “freeze then implement” process and keeps the legality checker authoritative.

### 2) Property-based tests: highest value properties + how to generate valid cases (Confidence: 0.84)

**Priority order (highest leverage first):**

1. **Determinism & replay invariants**

* Same seed + same initial state + same intents ⇒ identical event log (or identical deterministic subset if narration separated).
* RNG stream isolation (combat vs narration) if you implement streams.

2. **Legality monotonicity / fail-closed**

* If legality checker rejects an intent, resolver must not mutate state.
* If a required precondition is absent, intent cannot become legal by reordering modifiers.

3. **Modifier stacking algebra**

* Commutativity: order of application doesn’t change final bonus breakdown.
* Idempotence for same source where rules say “doesn’t stack.”
* Typed max/min behavior: “highest bonus / worst penalty” depending on your stacking rule definition.

4. **Resolution ordering contract**

* Your combat spec already asserts a strict order (legality → roll → gates → mitigation → reactions → update). Property tests should ensure **no operator reads post-update state** earlier than permitted, and triggers occur only at their defined stage. 

5. **Bounds / conservation**

* HP never exceeds max via “damage events.”
* Damage after mitigation ≥ 0.
* Duration ticking never goes negative.
* AoO counters never exceed allowed limits.

**Generating valid test cases**
Don’t try to generate “full D&D state” immediately. Generate **small, typed microstates** per subsystem:

* `ModifierSet` generator: produces (type, value, source, duration_tag) with constraints.
* `AttackScenario` generator: AB/AC bands, concealment flags, crit threat range, DR.
* `MovementScenario` generator: reach/threatened squares, AoO availability, path legality.

Then create a **“state builder”** that embeds those microstates into a minimal `WorldState` slice that passes schema validation.

**Shrinking**
Hypothesis shrinkers work best when your strategies are compositional and your objects are simple. Two pragmatic rules:

* Keep generated objects **small** (lists 0–6, not 0–50).
* Prefer enums + bounded ints over free text (your `source=st.text()` will create noisy failures; use a small symbol set).

**Runtime budget**
Set two lanes:

* `pytest -m quick`: sampled properties, < **60s** total
* `pytest -m slow`: deeper sampling / bigger ranges, < **10m** (nightly)

That’s enough to catch regressions without paralyzing iteration.

### 3) Performance regression detection: what to gate and how (Confidence: 0.82)

**Recommendation: 3-level enforcement**

1. **Hard budgets (CI blocking)**

* legality check
* core resolution primitives (attack/damage/save)
* full “monster turn micro-loop” (policy+legality+resolution) *only for deterministic policy modes / local models you actually ship*

2. **Soft budgets (warn-only)**

* narration latency
* TTS synthesis latency (because it’s hardware/model dependent)

3. **Trend tracking (always record)**

* record benchmarks with commit hash and machine fingerprint
* evaluate regression against rolling baseline (your 2-sigma idea is fine, but only meaningful if the runner is stable)

**Tolerance**

* For blocking budgets: **+15%** over baseline on stable runners (dedicated CI machine)
* For local/dev runners: **+25%** warning only

**Synthetic vs real scenarios**
Use both:

* **Synthetic microbenchmarks** for module primitives (tight, stable).
* **One or two real “golden” combat scenes** for end-to-end latency (captures integration cost).

This matches your requirement that the system be real-time voice playable and that legality/resolution remain fast.
