# RQ-NARR-001-A: Allowed Output Space + Truth Packet Interface

**Date:** 2026-02-11  
**Status:** Research Complete  
**System:** D&D 3.5e AI Dungeon Master - Spark Narration Subsystem  
**Target Model:** Qwen3 8B (8192 token context, expandable to 32K)

## Executive Summary

This document defines the **allowed output space** for Spark (narrative LLM) and specifies the **truth packet interface** that enables Spark to generate vivid, responsive narration while maintaining absolute mechanical fidelity to Box's authoritative game state.

**Core Principle:** Spark is a narrative renderer, not a rules adjudicator. It transforms Box-resolved mechanical outcomes into compelling D&D narration without ever inventing, contradicting, or adjudicating game mechanics.

## 1. Spark's Allowed Output Space Taxonomy

### 1.1 ALLOWED Output Categories (25 examples)

**A1. Descriptive Narration** - Attack hits, misses, saves, spell effects, deaths  
**A2. Atmospheric Scene-Setting** - Dungeons, combat tension, magic, weather  
**A3. NPC Dialogue** - Reactions, information, emotions, taunts  
**A4. Dramatic Embellishment** - Critical hits, fumbles, massive damage, victories  
**A5. Paraphrasing Box Receipts** - Margins, resistances, conditions, summaries

### 1.2 FORBIDDEN Output Categories (30 examples)

**F1. Mechanical Adjudication** - AoO rules, action economy, DCs, rule citations  
**F2. Contradicting Box** - Hit/miss reversals, damage minimization, condition denial  
**F3. Implying Different Reality** - HP misrepresentation, invented morale, invisible effects  
**F4. Retconning** - Scene contradictions, NPC reversals, inventory/environment changes  
**F5. Explaining Box Math** - Attack breakdowns, damage calcs, save bonuses  
**F6. Inventing Effects** - Environmental hazards, bonus effects, new conditions

### 1.3 GRAY AREAS (7 resolution rules)

**G1. Attack Margins** - Allowed with margin data, use for tone (not numbers)  
**G2. Wound Severity** - Allowed with HP%, scaled descriptions (not exact HP)  
**G3. Danger Assessment** - Atmospheric warnings OK, mechanical predictions forbidden  
**G4. Equipment Description** - Only if in truth packet  
**G5. Spell Sensory Details** - Pure flavor OK, mechanical implications forbidden  
**G6. NPC Reactions** - Visible events OK, hidden knowledge requires Box  
**G7. Tactical Positioning** - Only describe Box-confirmed positioning

## 2. Truth Packet Interface

### 2.1 Token Budget (8192 context)

- System Prompt: 1200 tokens (cached)
- Truth Packet: 800-1200 tokens
- Narration History: 2000 tokens
- Generation: 2500 tokens
- Safety Buffer: 1500 tokens

### 2.2 Schema Structure

```json
{
  "provenance_version": "1.0.0",
  "event": {
    "type": "attack_roll | spell_cast | ...",
    "timestamp": 3,
    "actor": {"name": "Thorin", "type": "PC"},
    "target": {"name": "Orc", "type": "MONSTER"},
    "details": {} // [BOX] event-specific data
  },
  "scene": {
    "location": {"name": "...", "terrain": "NORMAL", "lighting": "DIM"},
    "nearby_entities": [],
    "objects_of_interest": []
  },
  "conditions": {
    "actor_conditions": [],
    "target_conditions": []
  },
  "margins": { // [DERIVED] for tone calibration only
    "attack_margin": 4,
    "target_hp_percentage": 47.4,
    "is_critical": false
  },
  "narrative_context": {
    "recent_narrations": [...],
    "tone_directives": ["GRITTY", "TACTICAL"]
  },
  "restrictions": {
    "hidden_info": [],
    "max_length_tokens": 150
  }
}
```

### 2.3 Provenance Tags

- **[BOX]** - Absolute truth from game engine
- **[DERIVED]** - Calculated (use for tone, never cite directly)
- **[NARRATIVE]** - Prior narrations (maintain consistency)
- **[UNCERTAIN]** - Ambiguous (qualify with "seems")

**Error Handling:** Detect contradictions, fallback to templates, log violations  
**Gray Area Handling:** Margin thresholds, HP% bands, no tactical predictions
