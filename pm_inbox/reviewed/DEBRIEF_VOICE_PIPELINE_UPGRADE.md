## What Changed

### 1. Firewall / Proxy Fix (RESOLVED)

The long-standing CLI download failures (HuggingFace, GitHub, CivitAI timing out) were caused by Clash proxy (127.0.0.1:7890) not being picked up by CLI tools. The browser worked because it reads Windows system proxy settings; CLI tools need explicit env vars.

**Fix applied in four layers:**
- `~/.bashrc` — HTTP_PROXY/HTTPS_PROXY exports + proxy_on/proxy_off functions
- `pip.ini` — proxy setting for pip
- Windows user env vars — for CMD/PowerShell/GUI apps
- Git config — already had proxy (no change)

**AGENT_DEVELOPMENT_GUIDELINES Section 11a updated** with new proxy documentation.

### 2. Voice Reference Clip Upgrade: Kokoro → Piper

**Problem:** All 6 persona reference clips were generated by Kokoro (ONNX int8 on CPU). Kokoro's quantization artifacts — particularly on sibilants (S, SH, F sounds) — were cloned by Chatterbox, producing a "dusty record" quality on all voices.

**Solution:** Replaced all reference clips with Piper TTS output. Piper is a clean neural TTS engine (ONNX, CPU) with many distinct built-in voices and no quantization artifacts.

**Result:** Operator confirmed "noticeable improvement" on sibilant clarity across all personas. The dusty-record artifact is reduced (not eliminated — Chatterbox's vocoder has a quality ceiling on fricatives).

### 6. External Review (GPT cross-check, 2026-02-18)

Validated the Piper→Chatterbox pipeline as correct architecture. Key additions from review:
- **Speaker embedding similarity gate** recommended as preflight check (MFCC or speaker-encoder distance matrix, fail if any persona pair too close)
- **De-esser post-processing** on sibilant band only (light, persona-aware) for Chatterbox vocoder ceiling
- **Reference clip best practices confirmed:** 8-12s, single speaker, no music/reverb/breaths, normalized loudness, same text across personas for identity separation
- **Audio sub-pillar recommendations:**
  - SFX: Curated libraries first (Freesound/OpenGameArt), AudioLDM2 for compile-time gap-filling
  - Music: Curated packs first, MusicGen + loop-finding for infinite novelty later
  - Singing: Spoken verse + instrumental bed as MVP, true singing synthesis deferred
- Architecture validation: GT v12's "generate at worldgen/sessiongen, runtime selects by asset_id" is correct. Pin generator_id, generator_version, prompt, seed in canonical_request_bytes.
