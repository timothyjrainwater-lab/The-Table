## Summary

The engine currently references feats by string name in entity schemas (e.g., `"great_fortitude"`,
`"power_attack"`) but has no canonical table of feat benefits. Individual resolvers contain
hardcoded effects — but there's no authoritative source for "Great Fortitude gives +2 Fort" that
can be queried at runtime.

This WO ingests zellfaze feats.json (CC0, 221 verified feats with structured numeric benefits)
and populates a feat benefit registry at `aidm/data/feat_definitions.py`. No dependency on the
LST parser — zellfaze feats.json is clean JSON, direct read.

**No new engine behavior.** This WO creates the data layer. Engine WOs (Great Fortitude enforcement,
Power Attack routing, etc.) will query this registry in subsequent batches.

```json
  "name": "Great Fortitude",

  "prerequisites": [],
  "benefit": {

  }
}
```

- `fort`, `ref`, `will` — saving throw bonuses (int)
- `attack` — attack roll bonus (int)
- `ac` — armor class bonus (int)
- `damage` — damage bonus (int)
- Other fields may exist — document in debrief

```python
"""Feat benefit registry — populated from zellfaze feats.json (CC0).

Each entry maps a canonical feat ID (snake_case) to a FeatDefinition.
Source: zellfaze-zz/dnd-generator feats.json (CC0, 221 feats, verified 2026-02-26).
Facts are not copyrightable. This file contains only game mechanic values.
"""

from dataclasses import dataclass, field
from typing import Optional, Dict

@dataclass(frozen=True)
class FeatDefinition:
    feat_id: str            # snake_case canonical ID (e.g., "great_fortitude")
    name: str               # Display name (e.g., "Great Fortitude")
    feat_type: str          # "General" | "Metamagic" | "Item Creation" | other
    prerequisites: tuple    # Tuple of prereq strings (raw from source)
    fort_bonus: int = 0     # Saving throw bonus
    ref_bonus: int = 0
    will_bonus: int = 0
    attack_bonus: int = 0
    ac_bonus: int = 0
    damage_bonus: int = 0
    # Add fields as needed based on what's in zellfaze feats.json

FEAT_REGISTRY: Dict[str, FeatDefinition] = {
    "great_fortitude": FeatDefinition(
        feat_id="great_fortitude",
        name="Great Fortitude",
        feat_type="General",
        prerequisites=(),
        fort_bonus=2,
    ),
    # ... (221 entries total, generated from zellfaze feats.json)
}

def get_feat(feat_id: str) -> Optional[FeatDefinition]:
    return FEAT_REGISTRY.get(feat_id)
```
