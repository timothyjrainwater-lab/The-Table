### Deliverable 2: Mechanical-to-Semantics Mapping
Implemented as module-level functions in `semantics.py`:

| Function | Maps To |
|----------|---------|
| `map_delivery_mode()` | `DeliveryMode` enum — ray→BEAM, touch→TOUCH, self→SELF, burst→BURST_FROM_POINT, cone→CONE, line→LINE, emanation→EMANATION, spread→BURST_FROM_POINT, summoning→SUMMON, single+ranged→PROJECTILE |
| `map_staging()` | `Staging` enum — instantaneous→INSTANT, concentration→CHANNELED, round-duration→LINGER, burst+instant→TRAVEL_THEN_DETONATE |
| `map_scale()` | `Scale` enum — radius≥40/tier≥7→CATASTROPHIC, radius≥20/tier≥5→DRAMATIC, radius≥10/tier≥3→MODERATE, default→SUBTLE |
| `map_origin_rule()` | `OriginRule` enum |
| `map_vfx_tags()` | Tuple of VFX strings from damage_type, school, effect_type |
| `map_sfx_tags()` | Tuple of SFX strings from damage_type, effect_type |
| `is_active_feat()` | Filters feats — only trigger/grants_action feats included |
| `build_event_defaults()` | All 18 `EventCategory` values with default semantics |

**Note:** The dispatch specified `target_type == "ray"` → `RAY`, but `DeliveryMode` enum has no `RAY` value — only `BEAM`. Used `BEAM` as the correct mapping. Also handled `aoe_shape == "spread"` (present in real data but not in dispatch tables) as `BURST_FROM_POINT`.
