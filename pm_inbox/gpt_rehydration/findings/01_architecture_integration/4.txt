## FIX-001 (GAP-001)

### Objective

Eliminate permissive passthrough of unknown Box events at the Box→Lens boundary.

### Scope Boundaries

**IN:** Event contract registry, validation layer, Lens ingestion point, related tests.
**OUT:** Changing Box event types themselves.

### Concrete Artifacts

* `aidm/runtime/box_events.py`
* `aidm/lens/narrative_brief.py`
* Event contract registry (WO-046 reference)
* Related test modules (`test_box_event_contracts.py` if exists)

### Step Plan

1. Inventory all registered event types.
2. Identify current fallback behavior for unknown events.
3. Replace permissive dict passthrough with explicit failure or quarantined type.
4. Add boundary validator before NarrativeBrief assembly.
5. Ensure failure path is deterministic and includes event_type.
6. Update documentation (Box Event Contract section).
7. Run replay tests to confirm no legitimate events break.
8. Add regression tests.

### Acceptance Criteria

* Unknown event_type fails deterministically before Lens consumption.
* No dict-based event objects reach NarrativeBrief.
* Replay tests remain green.

### New Tests

* `test_unknown_event_rejected_at_boundary`
* `test_registered_event_passes_boundary_validation`

### Rollback Plan

Restore permissive passthrough with warning logs if regression reveals legitimate legacy event types.

### Stop Conditions

If existing save logs rely on passthrough events → escalate to define migration plan.

## FIX-002 (GAP-002)

### Scope Boundaries

**IN:** Prompt construction, token budgeting, validation wrapper.
**OUT:** Spark provider transport code.

* `aidm/spark/adapter.py`

* Prompt fixtures

### Step Plan

1. Define PromptPack v1 sections (Facts / Constraints / OutputSchema / Tone / SchemaVersion).
2. Formalize deterministic serialization order.
3. Implement schema_version field.
4. Replace direct string prompts with PromptPack assembly.
5. Integrate into SparkRequest.
6. Add truncation policy (deterministic).
7. Update documentation.
8. Add golden prompt tests.

### Acceptance Criteria

* Identical Box state → identical PromptPack string.
* Truncation does not change section ordering.
* Spark outputs validate against declared output schema.

### New Tests

* `test_promptpack_deterministic_serialization`
* `test_promptpack_truncation_stability`

### Rollback Plan

Fallback to legacy prompt path behind feature flag.

### Stop Conditions

If multiple prompt construction paths exist → halt and unify inventory first.

# 3) Highest-Risk Unknown

### Risk

Whether authoritative logs currently contain STT transcripts, timestamps, or narration artifacts.

If true, GAP-005 remediation could invalidate replay determinism.

### Fastest Resolution

1. Inspect one real authoritative JSONL event log from runtime.
2. Verify it contains only BoxEvents + deterministic intent artifacts.
3. Run replay twice with:

   * STT disabled
   * STT enabled with noise
     Compare replay hashes.

This is a 30–60 minute validation that determines whether determinism boundary is already compromised.

This plan preserves current architecture and converts boundary ambiguity into explicit, versioned contracts with measurable tests.
