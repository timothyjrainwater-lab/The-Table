## 1) 15 concrete failure scenarios (hostile, production-plausible)

> Legend: **Seam** = Box↔Lens / Lens↔Spark / Spark↔Immersion / Runtime orchestration.
> “Current likely behavior” is based on the **Gap Register** (esp. GAP-001..005). Where uncited, it’s an inference.

---

### S-01 — Unknown Box event type silently bypasses Lens validation

* **Inputs / sequence / environment:** Run a combat turn that produces a new/renamed `event_type` (e.g., `AOE_HIT_RESOLVED_V2`) due to a branch update, but Lens boundary registry wasn’t updated.
* **Seam / missing invariant:** **Box↔Lens**, **GAP-001** (unknown events passthrough).
* **Expected behavior:** **Fail-closed** at the boundary: explicit error naming `event_type`, halt narration/immersion for that turn.
* **Current likely behavior:** Unknown event passes as raw dict into downstream assembly; later steps crash or (worse) generate narration with partial/incorrect truth.
* **Detect it:** Boundary logs include “unknown event_type seen” OR the presence of untyped dict events in NarrativeBrief assembly.
* **Deterministic repro:** Add a fixture JSONL event containing `event_type="UNKNOWN_TEST_EVENT"` in the Box event stream and run the same turn seed twice.

---

### S-02 — Unknown event contains unexpected fields → nondeterministic narration differences

* **Inputs / sequence / environment:** Two runs with same seed; unknown dict event includes a python `set` or unordered mapping serialized differently between runs.
* **Seam / missing invariant:** **Box↔Lens**, **GAP-001**, determinism.
* **Expected behavior:** Boundary rejects unknown/unserializable event payloads deterministically.
* **Current likely behavior:** Prompt assembly order differs due to non-stable serialization, changing Spark output and breaking “same inputs → same narrated output” expectations (even if mechanics are identical).
* **Detect it:** Diff LensPromptPack strings or NarrativeBrief JSON across two runs with identical seed.
* **Deterministic repro:** Inject an unknown dict event whose fields include an unordered type or random key order; rerun twice.

---

### S-03 — Freeze timing bug: narration describes intermediate state (AoO / interrupts)

* **Inputs / sequence / environment:** Trigger AoO chain: movement provokes AoO, causes prone, interrupts a full attack; multi-event commit in one turn.
* **Seam / missing invariant:** **Box↔Lens**, **GAP-005** (log partition) + **freeze semantics undefined** (inferred).
* **Expected behavior:** Lens snapshot taken at **post-settlement** point (after all triggered events resolve) so narration matches final state.
* **Current likely behavior:** Lens captures mid-turn snapshot; narration says “the ogre stands” even though it ended prone, etc.
* **Detect it:** Compare narrated condition list vs final Box state hash; assert condition parity.
* **Deterministic repro:** Use a fixed seed + scripted encounter that reliably triggers AoO interrupt chain.

---

### S-04 — Containment leak: numeric grid coordinates or exact distances slip into narration

* **Inputs / sequence / environment:** Ranged attack with “target at 30 ft, partial cover,” includes exact squares or coordinates in Box truth.
* **Seam / missing invariant:** **Box↔Lens**/**Lens↔Spark**, **GAP-003** (containment matrix incomplete).
* **Expected behavior:** Distances mapped (“across the room”), coordinates never emitted.
* **Current likely behavior:** Some numeric fields leak into PromptPack/NarrativeBrief; Spark repeats them verbatim.
* **Detect it:** Regex scan prompts/narration for patterns like `\b\d+\s?(ft|feet|sq|x=|y=)\b`.
* **Deterministic repro:** Use a golden Box output fixture containing coords/distances; generate PromptPack and assert no forbidden patterns.

---

### S-05 — Containment leak: spell names / DC-like numbers imply mechanical authority

* **Inputs / sequence / environment:** Spell cast with a save DC and named spell (e.g., “Hold Person DC 17”).
* **Seam / missing invariant:** **Box↔Lens**/**Lens↔Spark**, **GAP-003**.
* **Expected behavior:** Either allow spell names but forbid DC numbers, or map both; policy must be explicit.
* **Current likely behavior:** Mixed leakage (spell names allowed ad hoc, DC leaks sometimes), causing “LLM is deciding” perception.
* **Detect it:** Prompt/narration scanner for `DC\s*\d+` and known spell list tokens if they’re meant to be forbidden.
* **Deterministic repro:** Fixed fixture: Box truth packet includes `dc=17, spell="Hold Person"`; verify mapping.

---

### S-06 — LensPromptPack drift: two call sites build different prompt formats

* **Inputs / sequence / environment:** Narration invoked from two different code paths (e.g., “end-of-turn narration” vs “combat receipt explanation”) that each build prompts differently.
* **Seam / missing invariant:** **Lens↔Spark**, **GAP-002** (no canonical PromptPack).
* **Expected behavior:** Single canonical PromptPack builder used everywhere.
* **Current likely behavior:** Inconsistent prompt layout; Grammar Shield behaves differently per call site; regressions look random.
* **Detect it:** Snapshot prompt strings for both paths and diff section headers/ordering.
* **Deterministic repro:** Two scripted runs that exercise both narration entrypoints with same underlying Box events; compare PromptPack serialization.

---

### S-07 — Token truncation drops constraints but keeps facts → hallucinated additions

* **Inputs / sequence / environment:** Very long scene context + memory; small token limit; truncation chops off “MUST NOT invent facts / fail-closed rules”.
* **Seam / missing invariant:** **Lens↔Spark**, **GAP-002** (token budget partition), **GAP-003**.
* **Expected behavior:** Constraints section is **always preserved**; facts truncated first in deterministic manner.
* **Current likely behavior:** Truncation is naive; constraints may be cut; Spark adds unsupported details.
* **Detect it:** PromptPack validator asserts required sections exist and are complete.
* **Deterministic repro:** Create an oversized PromptPack fixture with max_tokens set low; confirm truncation keeps constraints.

---

### S-08 — Spark returns non-JSON / malformed JSON; retry loop hangs or spikes latency

* **Inputs / sequence / environment:** Model returns “```json …” wrapper with trailing commentary; parser fails; retry prompt repeats.
* **Seam / missing invariant:** **Lens↔Spark**, **GAP-002** (output schema), **retry taxonomy missing** (inferred from contract).
* **Expected behavior:** Bounded retries (e.g., 1–2), then template fallback; deterministic error classification.
* **Current likely behavior:** Retry loop until timeout, blocking turn completion; or inconsistent fallback.
* **Detect it:** Metrics: retry_count, duration; alert on >N retries.
* **Deterministic repro:** Stub Spark provider to return malformed JSON consistently; run narration call.

---

### S-09 — Spark injects forbidden tokens (“HP=12”, “AC 18”, “x=4,y=9”) and Grammar Shield misses it

* **Inputs / sequence / environment:** Spark output includes explicit mechanical stats.
* **Seam / missing invariant:** **Lens↔Spark**, **GAP-003** containment enforcement beyond prompt-time; validator gaps.
* **Expected behavior:** Output rejected; fall back to template narration.
* **Current likely behavior:** Output accepted if validator only checks JSON parse; forbidden text slips through.
* **Detect it:** Post-generation scan of Spark output for forbidden patterns.
* **Deterministic repro:** Stub provider returns a known-bad output; ensure system blocks.

---

### S-10 — VRAM contention: Spark model resident + SDXL requested → crash or OOM

* **Inputs / sequence / environment:** GPU 12GB; Spark Qwen3 14B loaded; then an image request arrives; SDXL load fails.
* **Seam / missing invariant:** **Spark↔Immersion**, **GAP-004** (no ImmersionPlan), **GAP-002** (no structured directives), resource policy missing.
* **Expected behavior:** Deterministic eviction policy: unload one model; or skip image with explicit degrade result.
* **Current likely behavior:** Competing loads cause OOM, process crash, or long stall.
* **Detect it:** GPU OOM logs; adapter returns failure; watchdog catches stall.
* **Deterministic repro:** Use a config forcing both loads; request image generation immediately after Spark call.

---

### S-11 — Non-blocking rule violated: immersion retries block authoritative progression

* **Inputs / sequence / environment:** Image generation set to “retry until good”; critique gate fails repeatedly; turn waits.
* **Seam / missing invariant:** **Spark↔Immersion**, **GAP-004** (no bounded plan), **T3_IMMERSION_BOUNDARY** non-blocking requirement.
* **Expected behavior:** Immersion retries bounded; engine proceeds; placeholders used.
* **Current likely behavior:** Orchestrator waits on immersion completion because no plan enforces bounds.
* **Detect it:** Turn latency > threshold; engine “stuck” while immersion repeats.
* **Deterministic repro:** Stub image generator to always fail quality gate; run a turn requiring image.

---

### S-12 — Atmospheric artifacts accidentally enter authoritative log and alter replay hash

* **Inputs / sequence / environment:** STT transcript text, timestamps, or narration stored alongside Box events.
* **Seam / missing invariant:** **Runtime**, **GAP-005** (log partition).
* **Expected behavior:** Authoritative log contains only Box events + deterministic intent artifacts; atmospheric log separate and ignored by replay.
* **Current likely behavior:** Single log stream mixes both; replay hashes differ run-to-run due to timestamps/confidence.
* **Detect it:** Replay test fails when atmospheric varies; inspect log schema for transcript/time fields.
* **Deterministic repro:** Run same encounter twice with different STT confidence inputs; compare replay hash.

---

### S-13 — SessionBundle schema_version mismatch breaks loads with unclear errors

* **Inputs / sequence / environment:** Load older bundle/event log after a minor schema change; no migration path.
* **Seam / missing invariant:** **Integration/versioning**, **GAP-007**.
* **Expected behavior:** Deterministic, actionable error: “schema_version 1.0 unsupported; upgrade tool required”.
* **Current likely behavior:** Partial parse, KeyError deep in runtime, or silent data loss.
* **Detect it:** Loader validates schema_version early and fails loudly.
* **Deterministic repro:** Use a saved bundle fixture with bumped schema_version; attempt load.

---

### S-14 — Voice resolver jitter: same speaker gets different persona within session

* **Inputs / sequence / environment:** Multi-speaker scene; same character speaks across turns; Lens voice resolver uses Spark-described features.
* **Seam / missing invariant:** **Lens↔Spark / Spark↔Immersion**, **GAP-008**.
* **Expected behavior:** Within-session persona assignment invariant: `character_id → persona_id` stable.
* **Current likely behavior:** “deterministic enough” prompt drift yields different descriptors; mapping changes.
* **Detect it:** Assertion in tests: persona_id stable for same character across N turns.
* **Deterministic repro:** Stub Spark voice description to vary slightly; confirm mapping changes without cache policy.

---

### S-15 — Docs/process drift causes wrong dispatch: team reimplements “missing” work or refuses real work

* **Inputs / sequence / environment:** New agent reads older doctrine; sees “voice integration out-of-scope” and blocks fixes; or reads stale plan and reopens closed gaps.
* **Seam / missing invariant:** **Docs/hygiene**, **GAP-006**.
* **Expected behavior:** Single source of truth; doctrine errata makes scope explicit.
* **Current likely behavior:** Conflicting instructions lead to wrong priorities and audit noise.
* **Detect it:** Onboarding checklist references PSD as authority; doctool checks stale docs.
* **Deterministic repro:** Run onboarding checklist on a clean environment; see whether it directs to outdated constraints.

## Quick note on deterministic reproduction mechanics

To make these truly reproducible, each break test/run should standardize:

* **Fixed RNG seed** (Box)
* **Fixed fixture inputs** (event streams, PromptPack strings)
* **Stubbed Spark provider** (returns known outputs)
* **Stubbed immersion adapters** (fail/succeed deterministically)
* **Stable clocks** (freeze time or remove timestamps from authoritative artifacts)
